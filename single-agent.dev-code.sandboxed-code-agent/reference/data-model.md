# Data model — sandboxed-code-agent

Authoritative; `/akka:implement` produces these records and events verbatim.

| Record | Field | Type | Nullable | Meaning |

| | `request` | `Optional<ExecutionRequest>` | yes | Populated after `CodeSubmitted`. |

| | `cpuBudgetMs` | `long` | no | Budget in ms passed to dispatcher and monitor. |

Every nullable field on `Execution` is `Optional<T>`. The view's table updater wraps values with `Optional.of(...)`; callers use `.orElse(...)` or `.isPresent()` (Lesson 6).

## Enums

`SandboxBackend`: `DOCKER`, `E2B`, `MODAL`, `PLAYWRIGHT`.
`ScreeningDecision`: `APPROVED`, `BLOCKED`.
`ExecutionOutcome`: `COMPLETED`, `HALTED`, `FAILED`.
`ExecutionStatus`: `SUBMITTED`, `SCREENING`, `APPROVED`, `RUNNING`, `COMPLETED`, `HALTED`, `FAILED`.

## Events (`ExecutionEntity`)

| Event | Payload | Transition |
|---|---|---|
| `CodeSubmitted` | `request` | → SUBMITTED |
| `CodeGenerated` | `code` | → SCREENING |
| `CodeApproved` | `screening` (APPROVED) | → APPROVED |
| `CodeBlocked` | `screening` (BLOCKED) | stays in SCREENING (agent retries) |
| `ExecutionStarted` | — | → RUNNING |
| `ExecutionCompleted` | `output` | → COMPLETED (terminal happy) |
| `ExecutionHalted` | `haltReason` | → HALTED (terminal) |
| `ExecutionFailed` | `reason: String` | → FAILED (terminal) |

`emptyState()` returns `Execution.initial("")` with all `Optional` fields as `Optional.empty()` and `status = SUBMITTED`. `emptyState()` never references `commandContext()` (Lesson 3).

## View row

`ExecutionRow` mirrors `Execution`. The view holds the full row for the UI — there is no sensitive field to strip (unlike the docreview sample where `rawDocument` is audit-only). The generated code is displayed on the UI to give the operator full visibility into what the sandbox ran.

The view declares ONE query: `getAllExecutions: SELECT * AS executions FROM execution_view`. No `WHERE status = :status` filter — Akka cannot auto-index enum columns (Lesson 2); the endpoint filters client-side.

## Task definition (`ExecutionTasks.java`)

```java
public final class ExecutionTasks {
  public static final Task<ExecutionResult> EXECUTE_TASK = Task
      .name("Execute task")
      .description("Generate Python to solve the task description, call execute_code, and return an ExecutionResult")
      .resultConformsTo(ExecutionResult.class);

  private ExecutionTasks() {}
}
```

The companion `Tasks` class is mandatory for any `AutonomousAgent` (Lesson 7).

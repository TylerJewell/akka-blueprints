# Data model — meeting-facilitator

Authoritative; `/akka:implement` produces these records and events verbatim.

## Records

| Record | Field | Type | Nullable | Meaning |
|---|---|---|---|---|
| `TranscriptSegment` | `segmentId` | `String` | no | Unique id, generated by the poller. |
| | `sessionId` | `String` | no | Groups segments from the same meeting session. |
| | `speakerRaw` | `String` | no | Raw speaker display name; not yet sanitized. |
| | `textRaw` | `String` | no | Raw transcript text for the segment. |
| | `chatThreadRaw` | `String` | no | Raw chat messages during the segment. |
| | `capturedAt` | `Instant` | no | When the segment was captured. |
| `SanitizedSegment` | `redactedTranscript` | `String` | no | Speaker names and PII redacted. |
| | `redactedChat` | `String` | no | Chat PII redacted. |
| | `piiCategoriesFound` | `List<String>` | no | e.g., `["name","email","phone"]`. |
| `MeetingNotes` | `title` | `String` | no | 5–10 word title of the meeting's topic. |
| | `keyPoints` | `List<String>` | no | 3–6 one-sentence bullets. |
| | `actionItems` | `List<String>` | no | 0–4 concrete, role-scoped tasks. |
| | `generatedAt` | `Instant` | no | When the agent finished. |
| `ChatRecap` | `summary` | `String` | no | 1–3 sentences on chat activity. |
| | `messageCount` | `int` | no | Count of discrete chat messages. |
| | `generatedAt` | `Instant` | no | When the agent finished. |
| `GuardrailVerdict` | `passed` | `boolean` | no | true = publish, false = suppress. |
| | `reason` | `String` | no | One sentence naming the rule that fired (or "passed"). |
| `EvalResult` | `score` | `int` | no | 1–5 composite quality score. |
| | `rationale` | `String` | no | One sentence naming the strongest signal. |
| `MeetingSession` (entity state) | `sessionId` | `String` | no | — |
| | `rawSegment` | `TranscriptSegment` | no | Captured once at create; kept for audit. |
| | `sanitized` | `Optional<SanitizedSegment>` | yes | Populated after SegmentSanitized. |
| | `notes` | `Optional<MeetingNotes>` | yes | Populated after NotesSummarized. |
| | `chatRecap` | `Optional<ChatRecap>` | yes | Populated after ChatSummarized. |
| | `guardrailVerdict` | `Optional<GuardrailVerdict>` | yes | Populated after GuardrailPassed or GuardrailFailed. |
| | `evalScore` | `Optional<Integer>` | yes | Populated after EvalScored. |
| | `evalRationale` | `Optional<String>` | yes | Populated after EvalScored. |
| | `status` | `SessionStatus` | no | See enum. |
| | `createdAt` | `Instant` | no | When SegmentReceived emitted. |
| | `finishedAt` | `Optional<Instant>` | yes | Terminal state timestamp. |

## Enums

`SessionStatus`: `ACTIVE`, `SANITIZED`, `SUMMARIZED`, `PUBLISHED`, `SUPPRESSED`.

## Events (`MeetingSessionEntity`)

| Event | Payload | Transition |
|---|---|---|
| `SegmentReceived` | `rawSegment` | → ACTIVE |
| `SegmentSanitized` | `sanitized` | → SANITIZED |
| `NotesSummarized` | `notes` | (status stays SANITIZED; both agents must complete) |
| `ChatSummarized` | `chatRecap` | → SUMMARIZED (when both NotesSummarized + ChatSummarized present) |
| `GuardrailPassed` | `guardrailVerdict` | (no status change; sets verdict) |
| `GuardrailFailed` | `guardrailVerdict` | (no status change; sets verdict) |
| `SessionPublished` | — | → PUBLISHED (terminal) |
| `SessionSuppressed` | — | → SUPPRESSED (terminal) |
| `EvalScored` | `score, rationale` | (no status change; populates evalScore + evalRationale) |

## Events (`MeetingSessionQueue`)

| Event | Payload |
|---|---|
| `SegmentReceived` | `rawSegment` (the pre-sanitization form — used by the audit log only) |

## View row

`MeetingSessionRow` mirrors `MeetingSession` minus the raw `rawSegment` payload (the audit log retains that). The UI fetches full detail on-demand via `GET /api/sessions/{id}`.

# Data model — inbox-watcher

Authoritative; `/akka:implement` produces these records and events verbatim.

## Records

| Record | Field | Type | Nullable | Meaning |
|---|---|---|---|---|
| `IncomingMessage` | `messageId` | `String` | no | Unique id, generated by the inbox source. |
| | `from` | `String` | no | Raw sender; *not yet sanitized*. |
| | `subject` | `String` | no | Raw subject. |
| | `body` | `String` | no | Raw body. |
| | `receivedAt` | `Instant` | no | When the inbox source delivered it. |
| `SanitizedPayload` | `redactedSubject` | `String` | no | PII redacted. |
| | `redactedBody` | `String` | no | PII redacted. |
| | `piiCategoriesFound` | `List<String>` | no | e.g., `["email","phone","order-id"]`. |
| `ClassificationResult` | `kind` | `Classification` | no | One of `AUTO_DRAFTED`, `ESCALATE`, `IGNORE`. |
| | `confidence` | `String` | no | `"high" \| "medium" \| "low"`. |
| | `reason` | `String` | no | One short sentence. |
| `DraftedReply` | `subject` | `String` | no | `"Re: …"`. |
| | `body` | `String` | no | 3–5 paragraphs. |
| | `draftedAt` | `Instant` | no | When the agent finished. |
| `ApprovalDecision` | `approved` | `boolean` | no | true = SEND, false = DISMISS. |
| | `decidedBy` | `String` | no | Reviewer identifier. |
| | `reason` | `Optional<String>` | yes | Required when `approved=false`. |
| | `decidedAt` | `Instant` | no | When the human clicked. |
| `InboxMessage` (entity state) | `messageId` | `String` | no | — |
| | `incoming` | `IncomingMessage` | no | Captured once at create. |
| | `sanitized` | `Optional<SanitizedPayload>` | yes | Populated after MessageSanitized. |
| | `classification` | `Optional<ClassificationResult>` | yes | Populated after MessageClassified. |
| | `draft` | `Optional<DraftedReply>` | yes | Populated after ReplyDrafted. |
| | `decision` | `Optional<ApprovalDecision>` | yes | Populated after ReplyApproved/Rejected. |
| | `evalScore` | `Optional<Integer>` | yes | Populated after EvalScored. |
| | `evalRationale` | `Optional<String>` | yes | Populated after EvalScored. |
| | `status` | `MessageStatus` | no | See enum. |
| | `createdAt` | `Instant` | no | When MessageReceived emitted. |
| | `finishedAt` | `Optional<Instant>` | yes | Terminal state timestamp. |

## Enums

`Classification`: `AUTO_DRAFTED`, `ESCALATE`, `IGNORE`.
`MessageStatus`: `RECEIVED`, `SANITIZED`, `CLASSIFIED`, `DRAFTED`, `AWAITING_APPROVAL`, `SENT`, `DISMISSED`, `ESCALATED`.

## Events (`InboxMessageEntity`)

| Event | Payload | Transition |
|---|---|---|
| `MessageReceived` | `incoming` | → RECEIVED |
| `MessageSanitized` | `sanitized` | → SANITIZED |
| `MessageClassified` | `classification` | → CLASSIFIED (then conditional ESCALATE/DRAFTED/DISMISSED) |
| `MessageEscalated` | — | → ESCALATED (terminal) |
| `ReplyDrafted` | `draft` | → DRAFTED → AWAITING_APPROVAL |
| `ReplyApproved` | `decision` | (no status change) |
| `ReplyRejected` | `decision` | (no status change) |
| `MessageSent` | — | → SENT (terminal) |
| `MessageDismissed` | `reason` | → DISMISSED (terminal) |
| `EvalScored` | `score, rationale` | (no status change; populates evalScore + evalRationale) |

## Events (`MessageQueue`)

| Event | Payload |
|---|---|
| `MessageReceived` | `incoming` (the raw, pre-sanitization payload — used by the audit log) |

## View row

`InboxMessageRow` mirrors `InboxMessage` minus the raw `incoming` payload (the audit log keeps that). The UI fetches it on-demand via `GET /api/inbox/{id}`.

# Data model — aws-audit-monitor

Authoritative; `/akka:implement` produces these records and events verbatim.

## Records

| Record | Field | Type | Nullable | Meaning |
|---|---|---|---|---|
| `AwsFinding` | `findingId` | `String` | no | Unique id, generated by the scanner. |
| | `resourceType` | `String` | no | AWS resource type (e.g., `AWS::S3::Bucket`). |
| | `resourceArn` | `String` | no | Full resource ARN. May contain account ID segment. |
| | `ruleId` | `String` | no | Scanner rule identifier. |
| | `ruleName` | `String` | no | Human-readable rule name. |
| | `rawDetail` | `String` | no | Raw scanner finding detail. |
| | `detectedAt` | `Instant` | no | When the scanner detected the finding. |
| `NormalizedFinding` | `resourceType` | `String` | no | Same as raw; included for readability. |
| | `resourceArn` | `String` | no | ARN with account ID segment masked. |
| | `ruleId` | `String` | no | Same as raw. |
| | `controlDomain` | `String` | no | Mapped domain: `identity`, `data-protection`, `network`, `logging`, `encryption`. |
| | `normalizedDetail` | `String` | no | Detail with account-identifying segments stripped. |
| | `affectedAttributes` | `List<String>` | no | AWS resource attributes implicated (may be empty). |
| `AnalysisResult` | `severity` | `Severity` | no | One of `CRITICAL`, `HIGH`, `MEDIUM`, `LOW`, `INFO`. |
| | `controlRef` | `String` | no | Framework control id (e.g., `"CIS-2.1.5"`). |
| | `remediationSummary` | `String` | no | One paragraph. |
| | `confidence` | `String` | no | `"high" \| "medium" \| "low"`. |
| `CompiledReport` | `reportId` | `String` | no | Same id as `AuditReportEntity`. |
| | `executiveSummary` | `String` | no | Two paragraphs; max 250 words. |
| | `findingIds` | `List<String>` | no | Ordered by severity descending. |
| | `compiledAt` | `Instant` | no | When `ReportCompilerAgent` finished. |
| `ReviewDecision` | `published` | `boolean` | no | true = PUBLISH, false = DISMISS. |
| | `reviewedBy` | `String` | no | Risk officer identifier. |
| | `reason` | `Optional<String>` | yes | Required when `published=false`. |
| | `decidedAt` | `Instant` | no | When the risk officer clicked. |
| `AuditFinding` (entity state) | `findingId` | `String` | no | — |
| | `raw` | `AwsFinding` | no | Captured once at detection. |
| | `normalized` | `Optional<NormalizedFinding>` | yes | Populated after FindingNormalized. |
| | `analysis` | `Optional<AnalysisResult>` | yes | Populated after FindingAnalyzed. |
| | `reportId` | `Optional<String>` | yes | Populated after FindingCompiled. |
| | `decision` | `Optional<ReviewDecision>` | yes | Populated after FindingPublished/Dismissed. |
| | `evalScore` | `Optional<Integer>` | yes | Populated after EvalScored. |
| | `evalRationale` | `Optional<String>` | yes | Populated after EvalScored. |
| | `status` | `FindingStatus` | no | See enum. |
| | `createdAt` | `Instant` | no | When FindingDetected emitted. |
| | `finishedAt` | `Optional<Instant>` | yes | Terminal state timestamp. |
| `AuditReport` (entity state) | `reportId` | `String` | no | — |
| | `compiled` | `Optional<CompiledReport>` | yes | Populated after ReportCompilerAgent completes. |
| | `decision` | `Optional<ReviewDecision>` | yes | Populated after ReportPublished/Dismissed. |
| | `status` | `ReportStatus` | no | See enum. |
| | `createdAt` | `Instant` | no | When ReportCreated emitted. |
| | `publishedAt` | `Optional<Instant>` | yes | Terminal state timestamp. |

## Enums

`Severity`: `CRITICAL`, `HIGH`, `MEDIUM`, `LOW`, `INFO`.
`FindingStatus`: `DETECTED`, `NORMALIZED`, `ANALYZED`, `COMPILED`, `PENDING_REVIEW`, `PUBLISHED`, `DISMISSED`.
`ReportStatus`: `DRAFT`, `PENDING_REVIEW`, `PUBLISHED`, `DISMISSED`.

## Events (`AuditFindingEntity`)

| Event | Payload | Transition |
|---|---|---|
| `FindingDetected` | `raw` | → DETECTED |
| `FindingNormalized` | `normalized` | → NORMALIZED |
| `FindingAnalyzed` | `analysis` | → ANALYZED |
| `FindingCompiled` | `reportId` | → COMPILED → PENDING_REVIEW |
| `FindingPublished` | `decision` | → PUBLISHED (terminal) |
| `FindingDismissed` | `decision` | → DISMISSED (terminal) |
| `EvalScored` | `score, rationale` | (no status change; populates evalScore + evalRationale) |

## Events (`AuditReportEntity`)

| Event | Payload | Transition |
|---|---|---|
| `ReportCreated` | `reportId` | → DRAFT |
| `ReportFindingAdded` | `findingId` | (no status change; accumulates finding list) |
| `ReportPublished` | `compiled, decision` | → PUBLISHED (terminal) |
| `ReportDismissed` | `decision` | → DISMISSED (terminal) |

## Events (`FindingQueue`)

| Event | Payload |
|---|---|
| `FindingDetected` | `raw` (the raw, pre-normalization payload — used by the audit log) |

## View rows

`AuditFindingRow` mirrors `AuditFinding` minus `raw.rawDetail` (kept in the entity for full audit; excluded from the view for size). The UI fetches `rawDetail` on-demand via `GET /api/audit/findings/{id}`.

`AuditReportRow` mirrors `AuditReport`.

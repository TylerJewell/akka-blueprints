# Data model — live-eval-harness

Authoritative; `/akka:implement` produces these records and events verbatim.

## Records

| Record | Field | Type | Nullable | Meaning |
|---|---|---|---|---|
| `AgentDecision` | `decisionId` | `String` | no | Unique id, generated by the intake source. |
| | `agentId` | `String` | no | Identifier of the producing agent or service. |
| | `modelVersion` | `String` | no | Model version string at time of decision. |
| | `inputSummary` | `String` | no | Raw input context fed to the agent. |
| | `outputSummary` | `String` | no | Raw agent output. |
| | `userId` | `Optional<String>` | yes | User who triggered the decision; stripped before eval. |
| | `decidedAt` | `Instant` | no | When the agent produced the output. |
| `SanitizedDecision` | `sanitizedInput` | `String` | no | User-identifiable fields replaced with `[STRIPPED]`. |
| | `sanitizedOutput` | `String` | no | User-identifiable fields replaced with `[STRIPPED]`. |
| | `agentId` | `String` | no | Passed through unchanged. |
| | `modelVersion` | `String` | no | Passed through unchanged. |
| | `strippedFields` | `List<String>` | no | Names of fields that were stripped (e.g., `["userId"]`). |
| | `decidedAt` | `Instant` | no | From `AgentDecision`. |
| `RubricDimension` | `name` | `String` | no | Dimension name (e.g., `"accuracy"`). |
| | `score` | `int` | no | 1–5. |
| | `justification` | `String` | no | One sentence. |
| `EvalResult` | `decisionId` | `String` | no | Matches `AgentDecision.decisionId`. |
| | `dimensions` | `List<RubricDimension>` | no | Per-dimension breakdown. |
| | `overallScore` | `int` | no | Arithmetic mean of dimension scores, rounded. |
| | `summary` | `String` | no | One sentence naming the dominant signal. |
| | `evaluatedAt` | `Instant` | no | When scoring completed. |
| `EvalAlarmInfo` | `decisionId` | `String` | no | Which decision triggered the alarm. |
| | `score` | `int` | no | The score that crossed the threshold. |
| | `threshold` | `int` | no | The configured alarm threshold at time of firing. |
| | `firedAt` | `Instant` | no | When the alarm event was emitted. |
| `ScoreWindow` | `windowSizeDecisions` | `int` | no | Number of decisions in this window. |
| | `meanScore` | `double` | no | Mean `overallScore` across the window. |
| | `stdDev` | `double` | no | Standard deviation of `overallScore`. |
| | `meanPerDimension` | `Map<String,Double>` | no | Mean score per dimension name. |
| | `windowEnd` | `Instant` | no | Timestamp of the most recent decision in window. |
| `DriftAssessment` | `status` | `DriftStatus` | no | `OK`, `WATCH`, or `ALARM`. |
| | `narrative` | `String` | no | 1–2 sentences naming the signal. |
| | `flaggedDimensions` | `List<String>` | no | Dimension names below 3.0 mean. Empty when `OK`. |
| | `assessedAt` | `Instant` | no | When the agent returned this assessment. |
| `DecisionRecord` (entity state) | `decisionId` | `String` | no | — |
| | `incoming` | `AgentDecision` | no | Captured once at create; audit copy. |
| | `sanitized` | `Optional<SanitizedDecision>` | yes | Populated after `DecisionSanitized`. |
| | `evalResult` | `Optional<EvalResult>` | yes | Populated after `EvalScored`. |
| | `alarm` | `Optional<EvalAlarmInfo>` | yes | Populated after `EvalAlarm`. |
| | `status` | `DecisionStatus` | no | See enum. |
| | `createdAt` | `Instant` | no | When `DecisionReceived` emitted. |
| | `finishedAt` | `Optional<Instant>` | yes | Terminal state timestamp. |

## Enums

`DecisionStatus`: `RECEIVED`, `SANITIZED`, `EVALUATED`, `OK`, `ALARMED`.
`DriftStatus`: `OK`, `WATCH`, `ALARM`.

## Events (`DecisionEntity`)

| Event | Payload | Transition |
|---|---|---|
| `DecisionReceived` | `incoming: AgentDecision` | → RECEIVED |
| `DecisionSanitized` | `sanitized: SanitizedDecision` | → SANITIZED |
| `EvalScored` | `evalResult: EvalResult` | → EVALUATED |
| `EvalAlarm` | `alarm: EvalAlarmInfo` | EVALUATED → ALARMED (terminal) |
| `DecisionOk` | — | EVALUATED → OK (terminal) |

## Events (`DecisionQueue`)

| Event | Payload |
|---|---|
| `DecisionReceived` | `incoming: AgentDecision` (raw, pre-sanitization — audit log) |

## Events (`DriftSnapshotEntity`)

| Event | Payload | Effect |
|---|---|---|
| `DriftAssessed` | `assessment: DriftAssessment` | Updates current snapshot state. |
| `DriftAlarm` | `assessment: DriftAssessment` | Increments `alarmCount`; sets status=ALARM. |
| `DriftResolved` | `assessment: DriftAssessment` | Resets status to OK; `flaggedDimensions` cleared. |

## View rows

`EvalViewRow` mirrors `DecisionRecord` minus the raw `incoming.userId` field. The UI fetches the full record on demand via `GET /api/evals/{id}`.

`DriftSnapshot` (from `DriftSnapshotEntity`) surfaces `status`, `narrative`, `flaggedDimensions`, `lastAssessedAt`, `alarmCount`. Served via `GET /api/evals/drift` and included in the SSE `drift-update` event.

# Data model — open-loop-chaser

Authoritative; `/akka:implement` produces these records and events verbatim.

## Records

| Record | Field | Type | Nullable | Meaning |
|---|---|---|---|---|
| `SourceEvent` | `eventId` | `String` | no | Unique id generated by the simulator. |
| | `sourceType` | `SourceType` | no | One of `MEETING_NOTES`, `MESSAGE`, `DOCUMENT`. |
| | `rawText` | `String` | no | Raw source content; not yet sanitized. |
| | `rawAuthor` | `String` | no | Raw author identifier; not yet sanitized. |
| | `receivedAt` | `Instant` | no | When the source delivered it. |
| `SanitizedSource` | `redactedText` | `String` | no | PII redacted. |
| | `redactedAuthor` | `String` | no | PII redacted. |
| | `piiCategoriesFound` | `List<String>` | no | e.g., `["name","email","phone"]`. |
| `ExtractedActionItem` | `description` | `String` | no | Imperative-sentence description of the commitment. |
| | `suggestedOwner` | `String` | no | Role or placeholder from the sanitized text; `"unassigned"` if none. |
| | `dueDate` | `Optional<LocalDate>` | yes | Only set when the text contains an explicit date. |
| | `sourceEventId` | `String` | no | Back-reference to the originating `SourceEvent`. |
| `ExtractionResult` | `items` | `List<ExtractedActionItem>` | no | May be empty; never null. |
| | `extractionReason` | `String` | no | One sentence describing the extraction outcome. |
| `ComposedNudge` | `recipientHint` | `String` | no | Non-identifying role reference; no email or full name. |
| | `subject` | `String` | no | Reminder subject, ≤ 80 characters. |
| | `body` | `String` | no | 2–3 paragraphs. |
| | `composedAt` | `Instant` | no | When the agent finished. |
| `OwnerConfirmation` | `confirmedOwner` | `String` | no | Operator-confirmed recipient identifier. |
| | `confirmedBy` | `String` | no | Operator who confirmed. |
| | `confirmedAt` | `Instant` | no | When the operator confirmed. |
| `SnoozeRequest` | `snoozedBy` | `String` | no | Operator who snoozed. |
| | `snoozeDuration` | `Duration` | no | How long to suppress nudges. |
| | `snoozedAt` | `Instant` | no | When the snooze was applied. |
| `CloseRequest` | `closedBy` | `String` | no | Operator who closed. |
| | `closureNote` | `String` | no | Reason for closing. |
| | `closedAt` | `Instant` | no | When closed. |
| `ActionItem` (entity state) | `itemId` | `String` | no | — |
| | `sourceEventId` | `String` | no | — |
| | `sourceType` | `SourceType` | no | — |
| | `sanitized` | `SanitizedSource` | no | Set at creation. |
| | `extracted` | `ExtractedActionItem` | no | Set at creation. |
| | `ownerConfirmation` | `Optional<OwnerConfirmation>` | yes | Set after `OwnerConfirmed`. |
| | `lastNudge` | `Optional<ComposedNudge>` | yes | Most recent nudge; updated on each `NudgeComposed`. |
| | `snooze` | `Optional<SnoozeRequest>` | yes | Set after `ItemSnoozed`; cleared after `SnoozeLapsed`. |
| | `closure` | `Optional<CloseRequest>` | yes | Set after `ItemClosed`. |
| | `nudgeCount` | `int` | no | Increments on each `NudgeDispatched`. |
| | `status` | `ActionItemStatus` | no | See enum. |
| | `detectedAt` | `Instant` | no | When `ActionItemDetected` emitted. |
| | `finishedAt` | `Optional<Instant>` | yes | Terminal state timestamp. |

## Enums

`SourceType`: `MEETING_NOTES`, `MESSAGE`, `DOCUMENT`.

`ActionItemStatus`: `DETECTED`, `SANITIZED`, `PENDING`, `STALE`, `NUDGED`, `SNOOZED`, `CLOSED`, `FAILED`.

## Events (`ActionItemEntity`)

| Event | Payload | Transition |
|---|---|---|
| `ActionItemDetected` | `extracted, sanitized, sourceType` | → DETECTED |
| `ActionItemSanitized` | `sanitized` | → SANITIZED |
| `OwnerConfirmed` | `ownerConfirmation` | (no status change; populates ownerConfirmation) |
| `ItemMarkedStale` | — | → STALE |
| `NudgeComposed` | `lastNudge` | (no status change; updates lastNudge) |
| `NudgeDispatched` | — | → NUDGED; increments nudgeCount |
| `ItemSnoozed` | `snooze` | → SNOOZED |
| `SnoozeLapsed` | — | → PENDING |
| `ItemClosed` | `closure` | → CLOSED (terminal) |
| `ExtractionFailed` | `reason: String` | → FAILED (terminal) |

## Events (`SourceEventQueue`)

| Event | Payload |
|---|---|
| `SourceEventReceived` | `sourceEvent` (raw, pre-sanitization — the audit log) |

## View row

`ActionItemRow` mirrors `ActionItem` minus the raw `sourceEvent` fields (`rawText`, `rawAuthor`). The audit log retains those via `SourceEventQueue`. The UI fetches the full row via `GET /api/items/{id}`.

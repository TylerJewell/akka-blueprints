# Data model — long-term-memory-agent

Authoritative; `/akka:implement` produces these records and events verbatim.

## Records

| Record | Field | Type | Nullable | Meaning |
|---|---|---|---|---|
| `ConversationTurn` | `turnId` | `String` | no | UUID minted by `ConversationEndpoint`. |
| | `message` | `String` | no | The user's original message text. |
| | `reply` | `String` | no | The agent's reply text. Empty string until replied. |
| | `sentAt` | `Instant` | no | When the turn was added to the session. |
| | `repliedAt` | `Instant` | no | When the reply was recorded. `Instant.EPOCH` until replied. |
| | `status` | `TurnStatus` | no | Enum value. |
| `RawMemoryEntry` | `entryId` | `String` | no | UUID generated by the agent. |
| | `rawText` | `String` | no | Unredacted fact text. Never written to `MemoryEntity`. |
| | `kind` | `MemoryKind` | no | Enum value. |
| | `relevanceScore` | `float` | no | Agent's estimate of future usefulness (0.0–1.0). |
| `MemoryEntry` | `entryId` | `String` | no | Matches the originating `RawMemoryEntry.entryId`. |
| | `userId` | `String` | no | Owner of this memory. |
| | `sanitizedText` | `String` | no | PII-scrubbed form; stored and recalled. |
| | `piiCategoriesRedacted` | `List<String>` | no | e.g. `["email","ssn","person-name"]`. Empty if none. |
| | `kind` | `MemoryKind` | no | Enum value. |
| | `relevanceScore` | `float` | no | Preserved from `RawMemoryEntry`. |
| | `recordedAt` | `Instant` | no | When `MemoryEntity.storeMemory` was called. |
| | `expiresAt` | `Optional<Instant>` | yes | When the entry should be expired; `Optional.empty()` = no expiry. |
| `AgentReply` | `replyText` | `String` | no | The agent's response to the user message. |
| | `candidateMemories` | `List<RawMemoryEntry>` | no | Facts worth storing; may be empty. |
| | `generatedAt` | `Instant` | no | When the agent returned the reply. |
| `Session` (entity state) | `sessionId` | `String` | no | — |
| | `userId` | `String` | no | Owner of this session. |
| | `turns` | `List<ConversationTurn>` | no | Ordered by `sentAt`; max 50 entries. |
| | `status` | `SessionStatus` | no | See enum. |
| | `createdAt` | `Instant` | no | When `SessionCreated` emitted. |
| | `closedAt` | `Optional<Instant>` | yes | Terminal-state timestamp. |
| `UserMemoryStore` (entity state) | `userId` | `String` | no | — |
| | `memories` | `List<MemoryEntry>` | no | All stored entries for this user. |
| | `lastUpdatedAt` | `Instant` | no | Timestamp of the last `MemoryStored` or `MemoryExpired`. |

Every nullable field uses `Optional<T>`. The view table updater wraps values with `Optional.of(...)`; callers use `.orElse(...)` or `.isPresent()` (Lesson 6).

## Enums

`TurnStatus`: `PENDING`, `REPLIED`, `FAILED`.
`MemoryKind`: `FACT`, `PREFERENCE`, `RELATIONSHIP`, `CONTEXT`.
`SessionStatus`: `ACTIVE`, `CLOSING`, `CLOSED`.

## Events (`SessionEntity`)

| Event | Payload | Transition / Effect |
|---|---|---|
| `SessionCreated` | `userId: String` | → ACTIVE |
| `TurnAdded` | `turnId, message, sentAt` | Turn appended with status=PENDING; session stays ACTIVE |
| `ReplyRecorded` | `turnId, replyText, repliedAt` | Matching turn status → REPLIED |
| `MemoryExtracted` | `turnId, entries: List<RawMemoryEntry>` | No status change; triggers `MemorySanitizer` |
| `SessionClosed` | `closedAt` | → CLOSED |
| `TurnFailed` | `turnId, reason` | Matching turn status → FAILED; session stays ACTIVE |

`emptyState()` returns `Session.initial("", "")` with empty turns list, `status = ACTIVE`, `closedAt = Optional.empty()`. `emptyState()` never references `commandContext()` (Lesson 3).

## Events (`MemoryEntity`)

| Event | Payload | Effect |
|---|---|---|
| `MemoryStored` | `entry: MemoryEntry` | Entry appended to `memories`; `lastUpdatedAt` set |
| `MemoryExpired` | `entryId: String` | Matching entry removed from `memories`; `lastUpdatedAt` set |

`emptyState()` returns `UserMemoryStore.initial("")` with empty memories list.

## Views

### `SessionView` — row type `SessionRow`

Mirrors `Session`. The view's `getSessionsByUser` query filters by `userId`; the `getSession` query filters by `sessionId`. `turns` is included in full so the UI can render the conversation thread without a separate entity read.

### `MemoryView` — row type `MemoryRow`

Mirrors `MemoryEntry`. The view's `getMemoriesByUser` query filters by `userId` and orders by `recordedAt DESC`. `MemoryExtractionWorkflow.recallStep` queries this view to retrieve the top-K memories by `relevanceScore` for the given `userId`.

No `WHERE status` filter on either view — Akka cannot auto-index enum columns (Lesson 2); callers filter client-side where needed.

```java
      .name("Converse with memory")
      .description("Reply to the user's message using recalled memories, then extract new memory candidates")

```

The companion `Tasks` class is mandatory for any `AutonomousAgent` (Lesson 7).

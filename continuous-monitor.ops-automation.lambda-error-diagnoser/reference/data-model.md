## Enums

`Severity`: `LOW`, `MEDIUM`, `HIGH`, `CRITICAL`.

`IncidentStatus`: `DETECTED`, `NORMALISED`, `DIAGNOSED`, `EVALUATED`, `RESOLVED`, `DISMISSED`, `REOPENED`.

Severity inference rules used by `LogNormalizer`:
- `errorCategory = "oom"` → `HIGH`; if `coldStart = true` also → `CRITICAL`
- `errorCategory = "timeout"` and `coldStart = true` → `CRITICAL`; otherwise → `MEDIUM`
- `errorCategory = "dependency-failure"` → `HIGH`
- `errorCategory = "unhandled-exception"` → `MEDIUM`
- `errorCategory = "config-error"` → `LOW`

## Events (`IncidentEntity`)

| Event | Payload | Transition |
|---|---|---|
| `ErrorDetected` | `raw` | → DETECTED |
| `ErrorNormalised` | `normalised` | → NORMALISED |
| `DiagnosisProduced` | `diagnosis` | → DIAGNOSED |
| `DiagnosisEvaluated` | `eval` | → EVALUATED |
| `IncidentResolved` | — | → RESOLVED (terminal unless dismissed) |
| `IncidentDismissed` | `dismissReason, dismissedBy` | → DISMISSED |
| `IncidentReopened` | `reopenedBy` | → REOPENED |

## Events (`LogQueue`)

| Event | Payload |
|---|---|
| `ErrorDetected` | `raw` (the full `LambdaLogEntry` — used by the audit log) |

## View row

`IncidentRow` mirrors `Incident` minus the raw `stackTraceExcerpt` body (kept in `IncidentEntity` for the audit log; fetched on demand via `GET /api/incidents/{id}`). Every Optional field on the row uses `Optional<T>` — never a nullable reference.

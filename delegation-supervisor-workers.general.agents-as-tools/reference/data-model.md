# Data model

Every record the generated system defines. `Optional<T>` marks fields that are null before their lifecycle transition (Lesson 6). Akka's Jackson config serialises `Optional<T>` transparently — the wire value is the raw value or `null`.

## TaskRecord (entity state + view row)

| Field | Type | Nullable | Meaning |
|---|---|---|---|
| `taskId` | `String` | no | UUID, also the workflow id |
| `description` | `String` | no | The submitted task description |
| `status` | `TaskStatus` | no | Lifecycle state |
| `draft` | `Optional<DraftOutput>` | yes | WriterAgent output; null until available |
| `data` | `Optional<DataOutput>` | yes | DataAgent output; null until available |
| `result` | `Optional<AssembledResult>` | yes | Supervisor's assembled answer; null until `TaskCompleted` |
| `rejectionReason` | `Optional<String>` | yes | Set on `TaskRejected` or `TaskFailed` |
| `qualityScore` | `Optional<Integer>` | yes | 1–5; null until `QualityScored` |
| `qualityRationale` | `Optional<String>` | yes | One-line quality justification |
| `createdAt` | `Instant` | no | Task creation time |
| `finishedAt` | `Optional<Instant>` | yes | Set on any terminal transition |

The `TaskView` row type mirrors this with `Optional<T>` on every nullable field.

## Supporting records

```java
record TaskRequest(String description, String requestedBy) {}
record DraftOutput(String text, int wordCount, Instant draftedAt) {}
record NamedEntity(String name, String kind, String context) {}
record DataOutput(List<NamedEntity> entities, Instant summarisedAt) {}
record AssembledResult(String answer, List<String> toolsUsed, Instant assembledAt) {}
```

## Status enum

```java
enum TaskStatus { QUEUED, PROCESSING, COMPLETED, FAILED, REJECTED }
```

## Events

### TaskRecordEntity

| Event | Trigger |
|---|---|
| `TaskCreated` | Workflow creates the record (`createTask`) |
| `TaskProcessing` | Workflow transitions to processing before invoking supervisor |
| `TaskCompleted` | Supervisor returns an assembled result; stored in full |
| `TaskFailed` | Supervise step timed out; terminal |
| `TaskRejected` | Supervisor returned a rejection indicator; terminal |
| `QualityScored` | `QualitySampler` recorded a 1–5 score (self-transition on COMPLETED) |

### TaskQueue

| Event | Trigger |
|---|---|
| `TaskSubmitted` | `submitTask(description, requestedBy)` from endpoint or simulator |

Fields: `{ taskId, description, requestedBy, submittedAt }`.

## Notes on tool outputs

`DraftOutput` and `DataOutput` are intermediate payloads generated by `WriterAgent` and `DataAgent` respectively. The supervisor collects them internally during its tool-call loop, then bundles their essential content into `AssembledResult.answer`. Both intermediate records are also stored on `TaskRecord` for traceability, though the primary user-facing artifact is `AssembledResult`.

# Data model — cyber-guardian-agent

Authoritative; `/akka:implement` produces these records and events verbatim.

## Records

| Record | Field | Type | Nullable | Meaning |
|---|---|---|---|---|
| `ThreatSignal` | `signalId` | `String` | no | Unique id, generated by TelemetryPoller. |
| | `sourceHost` | `String` | no | Host that generated the signal. |
| | `category` | `String` | no | Raw category from the telemetry feed. |
| | `rawPayload` | `String` | no | Original event payload; kept in audit log only. |
| | `receivedAt` | `Instant` | no | When TelemetryPoller delivered it. |
| `ThreatAssessment` | `severity` | `Severity` | no | One of `LOW`, `MEDIUM`, `HIGH`, `CRITICAL`. |
| | `category` | `String` | no | MITRE ATT&CK tactic or normalised category. |
| | `reasoning` | `String` | no | One-sentence indicator summary. |
| | `confidenceScore` | `double` | no | 0.0–1.0 match confidence. |
| `RemediationPlaybook` | `steps` | `List<String>` | no | Ordered remediation actions. |
| | `estimatedEffort` | `String` | no | e.g., "2-4h", "30m". |
| | `generatedAt` | `Instant` | no | When the agent finished. |
| `HaltDirective` | `signalId` | `String` | no | Matches the incident. |
| | `isolatedHost` | `String` | no | Host targeted for isolation. |
| | `justification` | `String` | no | One-sentence rationale for the halt. |
| | `issuedAt` | `Instant` | no | When the directive was issued. |
| `LiftDecision` | `liftedBy` | `String` | no | Operator identifier. |
| | `reason` | `String` | no | Why the halt was lifted. |
| | `liftedAt` | `Instant` | no | When the operator confirmed. |
| `EvalEventPayload` | `signalId` | `String` | no | Matches the incident. |
| | `severity` | `Severity` | no | Final assessed severity. |
| | `category` | `String` | no | Final assessed category. |
| | `haltIssued` | `boolean` | no | Whether a HaltDirective was emitted. |
| | `playbookGenerated` | `boolean` | no | Whether a RemediationPlaybook was produced. |
| | `confidenceScore` | `double` | no | From ThreatAssessment. |
| | `closedAt` | `Instant` | no | When IncidentClosed was emitted. |
| `IncidentState` (entity state) | `signalId` | `String` | no | — |
| | `signal` | `ThreatSignal` | no | Captured once at create. |
| | `assessment` | `Optional<ThreatAssessment>` | yes | Populated after IncidentClassified. |
| | `halt` | `Optional<HaltDirective>` | yes | Populated after HaltIssued. |
| | `liftDecision` | `Optional<LiftDecision>` | yes | Populated after HaltLifted. |
| | `playbook` | `Optional<RemediationPlaybook>` | yes | Populated after PlaybookGenerated. |
| | `evalEvent` | `Optional<EvalEventPayload>` | yes | Populated after EvalEventEmitted. |
| | `status` | `IncidentStatus` | no | See enum. |
| | `createdAt` | `Instant` | no | When SignalReceived emitted. |
| | `closedAt` | `Optional<Instant>` | yes | Terminal state timestamp. |

## Enums

`Severity`: `LOW`, `MEDIUM`, `HIGH`, `CRITICAL`.

`IncidentStatus`: `RECEIVED`, `CLASSIFIED`, `HALTED`, `REMEDIATION`, `CLOSED`, `IGNORED`.

## Events (`IncidentEntity`)

| Event | Payload | Transition |
|---|---|---|
| `SignalReceived` | `signal` | → RECEIVED |
| `IncidentClassified` | `assessment` | → CLASSIFIED |
| `HaltIssued` | `halt` | → HALTED |
| `HaltLifted` | `liftDecision` | → REMEDIATION |
| `PlaybookGenerated` | `playbook` | (no status change; populates playbook field) |
| `IncidentClosed` | — | → CLOSED (terminal) |
| `EvalEventEmitted` | `evalEvent` | (no status change; populates evalEvent field) |

## Events (`TelemetryQueue`)

| Event | Payload |
|---|---|
| `SignalReceived` | `signal` (the raw ThreatSignal — audit log only) |

## View row

`IncidentRow` mirrors `IncidentState` minus `signal.rawPayload` (the audit log retains that). The UI fetches full incident detail on-demand via `GET /api/threats/{id}`. The list endpoint returns `IncidentRow` arrays only.

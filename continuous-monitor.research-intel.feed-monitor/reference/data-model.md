# Data model — feed-monitor

Authoritative; `/akka:implement` produces these records and events verbatim.

## Records

| Record | Field | Type | Nullable | Meaning |
|---|---|---|---|---|
| `FeedItem` | `itemId` | `String` | no | Unique id, generated by FeedPoller. |
| | `feedUrl` | `String` | no | Source feed URL. |
| | `title` | `String` | no | Item title from feed. |
| | `rawContent` | `String` | no | Full raw content body. |
| | `link` | `String` | no | Canonical URL for this item. |
| | `fetchedAt` | `Instant` | no | When FeedPoller fetched it. |
| `SummaryResult` | `summary` | `String` | no | 2–4 sentence AI summary. |
| | `topics` | `List<String>` | no | Up to 5 topic tags. |
| | `keyQuote` | `String` | no | Best verbatim sentence from rawContent. |
| | `summarizedAt` | `Instant` | no | When SummaryAgent finished. |
| `ClassificationResult` | `classification` | `FeedClassification` | no | One of `NOTIFY`, `SUPPRESS`, `REVIEW`. |
| | `confidence` | `String` | no | `"high" \| "medium" \| "low"`. |
| | `reason` | `String` | no | One short sentence. |
| | `targetChannel` | `Optional<String>` | yes | Set to channel when NOTIFY; null otherwise. |
| `SlackPost` | `channel` | `String` | no | Slack channel the message was sent to. |
| | `messageText` | `String` | no | Formatted message body. |
| | `postedAt` | `Instant` | no | When the stub was called. |
| | `guardrailPassed` | `boolean` | no | true = channel was in allow-list. |
| `EvalResult` | `score` | `int` | no | Integer 1–5. |
| | `rationale` | `String` | no | One-sentence explanation. |
| `FeedItemState` (entity state) | `itemId` | `String` | no | — |
| | `raw` | `FeedItem` | no | Captured once at create. |
| | `summary` | `Optional<SummaryResult>` | yes | Populated after ItemSummarized. |
| | `classification` | `Optional<ClassificationResult>` | yes | Populated after ItemClassified. |
| | `post` | `Optional<SlackPost>` | yes | Populated after ItemPosted. |
| | `evalScore` | `Optional<Integer>` | yes | Populated after EvalScored. |
| | `evalRationale` | `Optional<String>` | yes | Populated after EvalScored. |
| | `status` | `FeedItemStatus` | no | See enum. |
| | `createdAt` | `Instant` | no | When ItemReceived emitted. |
| | `finishedAt` | `Optional<Instant>` | yes | Terminal state timestamp. |

## Enums

`FeedClassification`: `NOTIFY`, `SUPPRESS`, `REVIEW`.

`FeedItemStatus`: `RECEIVED`, `SUMMARIZED`, `CLASSIFIED`, `NOTIFY_REQUESTED`, `POSTED`, `SUPPRESSED`, `PENDING_REVIEW`, `FAILED`.

## Events (`FeedItemEntity`)

| Event | Payload | Transition |
|---|---|---|
| `ItemReceived` | `raw` | → RECEIVED |
| `ItemSummarized` | `summary` | → SUMMARIZED |
| `ItemClassified` | `classification` | → CLASSIFIED |
| `NotifyRequested` | `classification.targetChannel` | → NOTIFY_REQUESTED |
| `ItemPosted` | `post` | → POSTED (terminal) |
| `ItemSuppressed` | — | → SUPPRESSED (terminal) |
| `ItemPendingReview` | — | → PENDING_REVIEW |
| `ItemFailed` | `reason: String` | → FAILED (terminal) |
| `ReviewOverridden` | `decidedBy, suppress: boolean, targetChannel` | (advances PENDING_REVIEW → NOTIFY_REQUESTED or SUPPRESSED) |
| `EvalScored` | `score, rationale` | (no status change; populates evalScore + evalRationale) |

## View row

`FeedItemRow` mirrors `FeedItemState` minus `raw.rawContent` (raw content is omitted from the view row to keep row size bounded; fetch via `GET /api/feed/{id}` when needed).

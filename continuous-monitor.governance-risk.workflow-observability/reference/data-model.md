# Data model — workflow-observability

Authoritative; `/akka:implement` produces these records and events verbatim.

## Records

| Record | Field | Type | Nullable | Meaning |
|---|---|---|---|---|
| `WorkloadItem` | `itemId` | `String` | no | Unique id, generated by the poller or caller. |
| | `workloadType` | `String` | no | Category hint: `policy-doc-review`, `log-anomaly-summary`, `incident-report`, `audit-trail-digest`. |
| | `payload` | `String` | no | Raw text content to be processed. |
| | `sourceTag` | `String` | no | Origin label (e.g., `siem-export`, `manual-submit`, `scheduler`). |
| | `receivedAt` | `Instant` | no | When the item entered the queue. |
| `TracingSpan` | `spanId` | `String` | no | Unique span id. |
| | `parentSpanId` | `String` | no | Root span id or parent step span id. |
| | `operationName` | `String` | no | e.g., `RouterAgent.route`, `SummaryAgent.process`. |
| | `startedAt` | `Instant` | no | Span start. |
| | `finishedAt` | `Instant` | no | Span end. |
| | `durationMs` | `long` | no | `finishedAt - startedAt` in milliseconds. |
| | `promptTokens` | `int` | no | Tokens in the LLM prompt. |
| | `completionTokens` | `int` | no | Tokens in the LLM completion. |
| | `status` | `SpanStatus` | no | `OK`, `ERROR`, or `TIMEOUT`. |
| | `errorMessage` | `String` | yes | Non-null only when `status != OK`. |
| `RoutingDecision` | `category` | `RoutingCategory` | no | `SUMMARISE`, `ESCALATE`, or `SKIP`. |
| | `confidence` | `String` | no | `"high" \| "medium" \| "low"`. |
| | `rationale` | `String` | no | One short sentence. |
| `SummaryResult` | `summary` | `String` | no | 2-4 sentence summary. |
| | `keyFindings` | `String` | no | 1-3 bullet points as newline-separated string. |
| | `outputTokens` | `int` | no | Estimated tokens produced. |
| | `processedAt` | `Instant` | no | When SummaryAgent finished. |
| `EvalResult` | `score` | `int` | no | 1-5. |
| | `rationale` | `String` | no | One sentence. |
| | `evaluatedAt` | `Instant` | no | When EvalSampler ran. |
| `WorkloadItemState` (entity state) | `itemId` | `String` | no | — |
| | `item` | `WorkloadItem` | no | Captured once at registration. |
| | `routing` | `Optional<RoutingDecision>` | yes | Populated after ItemRouted. |
| | `result` | `Optional<SummaryResult>` | yes | Populated after ItemProcessed. |
| | `spans` | `List<TracingSpan>` | no | Grows with each SpanEmitted; never null (starts empty). |
| | `eval` | `Optional<EvalResult>` | yes | Populated after EvalCompleted. |
| | `status` | `ItemStatus` | no | See enum. |
| | `createdAt` | `Instant` | no | When ItemRegistered emitted. |
| | `completedAt` | `Optional<Instant>` | yes | Terminal state timestamp. |

## Enums

`RoutingCategory`: `SUMMARISE`, `ESCALATE`, `SKIP`.
`SpanStatus`: `OK`, `ERROR`, `TIMEOUT`.
`ItemStatus`: `QUEUED`, `ROUTED`, `PROCESSING`, `EXPORTED`, `EVALUATED`, `ESCALATED`, `SKIPPED`, `FAILED`.

## Events (`WorkloadItemEntity`)

| Event | Payload | Transition |
|---|---|---|
| `ItemRegistered` | `item` | -> QUEUED |
| `ItemRouted` | `routing` | -> ROUTED |
| `ItemProcessed` | `result` | -> PROCESSING |
| `SpanEmitted` | `span` | (no status change; appends to spans list) |
| `TracesExported` | — | -> EXPORTED |
| `EvalCompleted` | `eval` | -> EVALUATED |
| `ItemEscalated` | — | -> ESCALATED (terminal) |
| `ItemSkipped` | — | -> SKIPPED (terminal) |
| `ItemFailed` | `errorMessage` | -> FAILED (terminal) |

## Events (`WorkloadQueue`)

| Event | Payload |
|---|---|
| `WorkloadItemReceived` | `item` (the original item — the audit record) |

## View row

`WorkloadItemRow` mirrors `WorkloadItemState` in full, including the `spans` list. The `/api/traces` list endpoint returns rows without the `spans` field for performance; the full span list is available via `/api/traces/{id}/spans` and `/api/traces/{id}`.
